# P2437 蜜蜂路线

## 题目描述

一只蜜蜂在下图所示的数字蜂房上爬动,已知它只能从标号小的蜂房爬到标号大的相邻蜂房,现在问你：蜜蜂从蜂房 $m$ 开始爬到蜂房 $n$，$m<n$，有多少种爬行路线？（备注：题面有误，右上角应为 $n-1$）

![](https://cdn.luogu.com.cn/upload/pic/1575.png)

## 输入格式

输入 $m,n$ 的值。

## 输出格式

爬行有多少种路线。

## 输入输出样例 #1

### 输入 #1

```
1 14
```

### 输出 #1

```
377
```

## 说明/提示

对于 $100\%$ 的数据，$1 \le m,n\le 1000$。

## 解析

对于这题,我在integrity&anti corruption课上看了2分钟就知道了要用递推,而且是毫无技术含量的斐波那契递推,但是就当我兴冲冲写好提交之后,拿了40分,我以为是我那个边界条件没有考虑到,于是我又加上了当n=2或m+1的时候,直接输出1

但是就算如此还是40分,看来问题不在这个边界条件上,我一筹莫展,不应该啊,我看了讨论版,都在说高精度

斐波那契增长速度很快,1000以内会膨胀到long long都装不下,只能用高精度了

main函数的逻辑很简单,就是把每个点的方法数拆解成能够一步到达这个点的方法数之和,就可以递推(梦回高考的那个走格子)

我卡在那呢,就是高精度加法里面有的细节我给忽略了

#### 1.

carry的初始值是0而不是1,我一开始设置成1了,这就导致最开始的时候多进了1

#### 2.

2.最致命的地方,i的循环范围,一开始我int lim=max(a.size(),b.size())

然后i循环到较短数组的尾部的时候,由于我是max,所以会继续下去,然后这个时候就越界了,然后就会出现一些不可名状的奇怪大数字

正确做法当如AC代码那样,哪个数组还有数,就加上,到尾了,就不加了

#### 3.

3.循环结束后,检查carry是不是还有数字,如果有,一定要push!!这个也容易忘记

#### 4.

4.函数签名的地方,vector<int> &a,这个&很关键,我说白了,这个漏掉是真的糖币
