# P1255 数楼梯 - 从递归到递推的优化总结

## 一、题目描述

**题目**：P1255 数楼梯
**要求**：计算N阶楼梯的走法总数（一步上一阶或两步上二阶）
**数据范围**：1 ≤ N ≤ 5000
**特点**：结果极大，需要高精度处理

## 二、递归解法及局限性

### 递归代码

cpp

```
#include <bits/stdc++.h>
using namespace std;
long long ans = 0;

void pa(long long start, long long n) {
    if(start == n) {
        ans++;
        return;
    }
    else if(start > n) {
        return;
    }
    pa(start+1, n);
    pa(start+2, n);
}

int main() {
    long long n;
    cin >> n;
    pa(0, n);
    cout << ans;
    return 0;
}
```

### 递归的局限性

1. **时间复杂度过高**：O(2ⁿ)，指数级增长

   - n=50时，需要计算约 2⁵⁰ ≈ 1千万亿次

2. **重复计算严重**

   ```
   ├─ f(4)
   │  ├─ f(3)
   │  │  ├─ f(2)  ← 重复计算

   │  └─ f(2)     ← 重复计算
   └─ f(3)        ← 重复计算
      ├─ f(2)     ← 重复计算
      └─ f(1)
   ```

3. **数据类型溢出**：long long无法存储超大结果

   - n=50时结果约 2⁵⁰ ≈ 1.1e15（刚好在范围内）
   - n=5000时结果有1000多位，远超long long范围

### 为什么会超时

- 题目n最大5000，2⁵⁰⁰⁰是天文数字
- 即使每秒计算1亿次，也需要宇宙年龄的时间
- 递归树节点数呈指数增长，无法在时限内完成

## 三、递推优化解法

### 递推代码（含高精度）

cpp

```
#include <bits/stdc++.h>
using namespace std;

// 高精度加法
vector<int> add(vector<int> &a, vector<int> &b) {
    int carry = 0;
    int len = max(a.size(), b.size());
    vector<int> c;
    
    for(int i = 0; i < len; i++) {
        if(i < a.size()) carry += a[i];
        if(i < b.size()) carry += b[i];
        c.push_back(carry % 10);
        carry /= 10;
    }
    
    if(carry) {
        c.push_back(carry);
    }
    return c;
}

int main() {
    int n;
    cin >> n;
    
    // 处理边界情况
    if(n == 0) {
        cout << 1 << endl;
        return 0;
    }
    
    // dp[i]表示到第i阶的方法数（用vector存储大数）
    vector<vector<int>> dp(n + 1);
    
    // 初始化
    dp[0] = {1};  // 0阶：1种方法（不动）
    dp[1] = {1};  // 1阶：1种方法（走1步）
    
    // 递推计算：dp[i] = dp[i-1] + dp[i-2]
    for(int i = 2; i <= n; i++) {
        dp[i] = add(dp[i-1], dp[i-2]);
    }
    
    // 倒序输出（个位在索引0）
    for(int i = dp[n].size() - 1; i >= 0; i--) {
        cout << dp[n][i];
    }
    
    return 0;
}
```

### 递推的优越性

1. **时间复杂度优化**：O(n × L) ≈ O(5000 × 1000) = 5×10⁶次运算
   - 相比递归的O(2ⁿ)，效率提升无数倍
2. **无重复计算**：每个状态只计算一次
   - 用数组保存中间结果
   - 后一个状态由前两个状态推出
3. **空间可控**：O(n × L) ≈ 5000 × 1000 ≈ 5MB内存
4. **可处理大数**：用vector存储数字的每一位，突破整数类型限制

### 递推思路详解

**核心思想**：从已知推向未知，用已求出的解推导新解

**状态定义**：`dp[i]` = 走到第i阶楼梯的方法数

**状态转移方程**：

text

```
dp[i] = dp[i-1] + dp[i-2]
```

- 走到第i阶可以从第i-1阶走1步到达
- 也可以从第i-2阶走2步到达
- 所以方法数等于两者之和

**初始化**：

text

```
dp[0] = 1  // 0阶：1种方法（不动）
dp[1] = 1  // 1阶：1种方法（直接走1步）
```

**计算过程示例**（n=4）：

text

```
dp[0] = 1
dp[1] = 1
dp[2] = dp[1] + dp[0] = 1 + 1 = 2
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
dp[4] = dp[3] + dp[2] = 3 + 2 = 5
结果：5种走法
```

**大数存储**：每位数字倒序存储在vector中

text

```
数字12345  →  vector[5,4,3,2,1]
          索引0是个位，方便进位处理
```

## 四、经验总结

### 1. **算法选择的重要性**

- 递归适合：问题规模小，代码简洁直观
- 递推动态规划适合：大规模问题，有重叠子问题

### 2. **时间复杂度分析能力**

- 遇到递归先分析复杂度
- 指数级算法（O(2ⁿ)）必须优化
- 观察数据范围，选择合适算法

### 3. **数据类型意识**

- 普通整数类型有范围限制（int约2e9，long long约9e18）
- 结果可能极大时，考虑高精度或取模

### 4. **递推动态规划的关键**

- **状态定义**：明确dp[i]代表什么
- **状态转移**：找出前后关系
- **初始化**：正确设置初始值
- **计算顺序**：从小规模向大规模

### 5. **本题核心收获**

- **递归到递推的思维转变**：从"怎么到终点"到"终点怎么来"
- **高精度实现技巧**：vector存储，倒序处理，逐位相加
- **空间优化思路**：可以只用三个变量滚动计算（本题未用）

### 6. **类似题目拓展**

- 斐波那契数列
- 爬楼梯变种（一步可走1/2/3阶）
- 不同路径问题
- 杨辉三角

**总结**：本题完美展示了算法优化的重要性——从无法运行的递归，到高效通过的递推+高精度，关键在于选择合适的算法和数据结构！
