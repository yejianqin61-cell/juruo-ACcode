# 题目

# P1028 [NOIP 2001 普及组] 数的计算

## 题目描述

给出正整数 $n$，要求按如下方式构造数列：

1. 只有一个数 $n$ 的数列是一个合法的数列。
2. 在一个合法的数列的末尾加入一个正整数，但是这个正整数不能超过该数列最后一项的一半，可以得到一个新的合法数列。

请你求出，一共有多少个合法的数列。两个合法数列 $a, b$ 不同当且仅当两数列长度不同或存在一个正整数 $i \leq |a|$，使得 $a_i \neq b_i$。

## 输入格式

输入只有一行一个整数，表示 $n$。

## 输出格式

输出一行一个整数，表示合法的数列个数。

## 输入输出样例 #1

### 输入 #1

```
6

```

### 输出 #1

```
6

```

## 说明/提示

### 样例 1 解释

满足条件的数列为：
- $6$
- $6, 1$
- $6, 2$
- $6, 3$
- $6, 2, 1$
- $6, 3, 1$

### 数据规模与约定

对于全部的测试点，保证 $1 \leq n \leq 10^3$。

# 二傻子纯递归代码



```c++
#include <bits/stdc++.h>
using namespace std;
vector<int> num;
int ans=1;
void dfs(){
	
	int index=num.size();
	int lim=num[index-1]/2;
	for(int i=1;i<=lim;i++){
		ans++;
		num.push_back(i);
		dfs();
		num.pop_back();
	}
}

int main()
{
	int n;
	cin>>n;
	num.push_back(n);
	
	dfs();
	cout<<ans;
	return 0;
}
```
先来说说这个二傻子纯递归代码把,这是我一次性写出来的,能够不借助AI帮助,从头到尾想明白暴力递归的代码并写出来,也算是我这个蒟蒻的一种进步了,这也是蓝桥惨败之后的痛定思痛,不能一卡住就问AI,要自己想了足够久,实在便秘,才能问问思路.......

思路就是,全局变量一个ans记录总数列个数,num数组记录当前数列状态,然后就是dfs,从1一直搜索到lim,在压入之前,ans++,然后继续dfs,到下一层继续搜索,由于压入栈的数越来越小,直到lim足够小导致无法进入for循环,而直接返回,这个时候,一条链路就走到头了,就开始回溯了,回溯的时候,要记得popback出来,然后继续for循环,步进...直到第一层搜索循环到了头儿,就会返回到调用它的地方,也就是main函数,这个时候ans就是答案了.

但是很显然,蒟蒻之所以是蒟蒻,就是因为算法太蠢,1e3的数据规模,1s的时间限制,是遭不住你这么暴力递归的...

不足之处在于,在于比如说:当前是15, 6 ,3,2,1

好我等一下到了15,7,3的地步的时候,我又去吭哧吭哧找以3开头的数列个数了,明明之前已经找过3为首的数列个数了,你又一个个ans++,这种把二叉树走遍的方法肯定不行,;总结起来就是说,你在每次搜索到一个last的时候,你又要去探究人家last背后的身世,如果有一种方法,能够让调查过的last自己带着自己的方法数,这样就不会反复搜索了



# DP动态规划



于是,我们使用unordered_map,记忆化搜索!

### 代码




```c++
#include <bits/stdc++.h>
using namespace std;
#define int long long
int n;
unordered_map <int,int>mp;
int dfs(int last){
	if(mp.count(last)){
		return mp[last];
	}
	int count=1;
	for(int i=1;i<=last/2;i++){
		if(mp.count(i)!=0){
			count+=mp[i];
		else{
			int temp=dfs(i);
			mp[i]=temp;
			count+=mp[i];
		}
	}
	return count;
}

signed main()
{
	cin>>n;
	int answer=dfs(n);	
	cout<<answer;
	return 0;
}
```
很快奥,咱也是用上哈希表了......

两个版本的差距在于记忆化搜索...

dfs思路:首先肯定是先检查last之前有没有记录下来过,如果有,那就直接return,没有就继续....这个是精髓,没有它就没有所谓的记忆化;

然后规定count=1;因为[last]本身也是个合法的数组,然后开始遍历,从1一直到last/2,如果i有被记录过,也是直接加到count上面了,如果没有,就把i作为last,跳脱到另一个平行宇宙进行搜索,一番折腾之后,它会带着返回值回来,赋予mp[i] (这个也是必不可少的!!!),然后加到count上

每个dfs(last)只有一个return,return的是count,也就是以last为首的数列数目,

逐步回溯之后,一定会回到属于n的count!!!!
