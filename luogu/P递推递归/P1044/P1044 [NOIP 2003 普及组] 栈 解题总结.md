# P1044 [NOIP 2003 普及组] 栈 解题总结

## 一、题目回顾

给定一个操作数序列 $1,2,\ldots,n$，通过入栈（push）和出栈（pop）操作，可以得到不同的输出序列。求可能得到的输出序列的总数。

**数据范围**：$1 \leq n \leq 18$

**示例**：$n=3$ 时，输出序列总数为 $5$

------

## 二、AC的递归代码

cpp

```
#include <bits/stdc++.h>
using namespace std;

int dfs(int inStack, int nextNum, int n) {
    // inStack: 栈中元素个数
    // nextNum: 下一个要入栈的数字
    
    // 所有数字都已处理完毕
    if (nextNum > n) {
        return 1;  // 只有一种方式：把栈中所有元素弹出
    }
    
    int count = 0;
    
    // 操作1：如果还有数字可以入栈，则入栈
    if (nextNum <= n) {
        count += dfs(inStack + 1, nextNum + 1, n);
    }
    
    // 操作2：如果栈不为空，则出栈
    if (inStack > 0) {
        count += dfs(inStack - 1, nextNum, n);
    }
    
    return count;
}

int main() {
    int n;
    cin >> n;
    
    int ans = dfs(0, 1, n);
    cout << ans << endl;
    
    return 0;
}
```

## 三、递归的局限性

1. **时间复杂度高**：$O(2^n)$，当 $n=18$ 时勉强能过，但 $n$ 再大就会超时
2. **重复计算**：相同的状态 $(inStack, nextNum)$ 会被多次计算
3. **无法处理更大数据**：对于 $n>20$ 的问题，递归会无法在合理时间内完成

**改进方向**：可以加上记忆化搜索，将 $(inStack, nextNum)$ 的状态存下来，避免重复计算

## 四、AC的递推代码

### 代码一：二维DP（正向思维，最符合常理）

cpp

```
#include <bits/stdc++.h>
using namespace std;

int dp[20][20];  // dp[i][j] 表示还有i个数没入栈，栈里有j个数

int main() {
    int n;
    cin >> n;
    
    dp[n][0] = 1;  // 初始状态：还有n个数没入栈，栈为空
    
    // 从大到小遍历，因为入栈操作会让i变小
    for (int i = n; i >= 0; i--) {
        for (int j = 0; j <= n; j++) {
            if (dp[i][j] == 0) continue;
            
            // 入栈：从"还有i个数没入栈"变成"还有i-1个数没入栈"
            if (i > 0) {
                dp[i-1][j+1] += dp[i][j];  // 消耗一个待入栈的数，栈中增加一个数
            }
            
            // 出栈：栈中减少一个数
            if (j > 0) {
                dp[i][j-1] += dp[i][j];    // 待入栈的数不变，栈中弹出一个数
            }
        }
    }
    
    // 最终状态：没有数待入栈(i=0)，栈为空(j=0)
    cout << dp[0][0] << endl;
    
    return 0;
}
```

### 代码二：二维DP（反向思维，常见写法）

cpp

```
#include <bits/stdc++.h>
using namespace std;

int dp[20][20];  // dp[i][j] 表示还有i个数没入栈，栈里有j个数

int main() {
    int n;
    cin >> n;
    
    dp[0][0] = 1;  // 基础状态
    
    for (int i = 0; i <= n; i++) {
        for (int j = 0; j <= n; j++) {
            if (i == 0 && j == 0) continue;
            
            // 从过去的状态累加到当前状态
            if (i > 0) {
                dp[i][j] += dp[i-1][j+1];  // 通过入栈来到现在
            }
            if (j > 0) {
                dp[i][j] += dp[i][j-1];    // 通过出栈来到现在
            }
        }
    }
    
    cout << dp[n][0] << endl;
    return 0;
}
```

### 代码三：卡特兰数（最简洁）

cpp

```
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    long long catalan[20] = {0};
    catalan[0] = 1;
    
    for (int i = 1; i <= n; i++) {
        catalan[i] = catalan[i-1] * (4*i - 2) / (i + 1);
    }
    
    cout << catalan[n] << endl;
    return 0;
}
```

## 五、递推的思想

### 1. 状态定义

`dp[i][j]` 表示 **还有i个数没入栈，栈里有j个数**

### 2. 两种操作对应两种转移

**正向思维（从当前状态推向未来）**：

- **入栈**：`dp[i][j] → dp[i-1][j+1]`
  - 消耗一个待入栈的数，栈中增加一个数
  - 条件：还有数可以入栈（`i > 0`）
- **出栈**：`dp[i][j] → dp[i][j-1]`
  - 待入栈的数不变，栈中弹出一个数
  - 条件：栈不为空（`j > 0`）

**反向思维（从过去累加到当前）**：

- **入栈得来**：`dp[i][j] += dp[i-1][j+1]`
- **出栈得来**：`dp[i][j] += dp[i][j-1]`

### 3. 初始状态和目标状态

- **初始状态**：`dp[n][0] = 1`（还有n个数没入栈，栈为空）
- **目标状态**：`dp[0][0]`（没有数待入栈，栈为空）

## 六、总结经验

### 1. 递归 vs 递推

- **递归**：直观模拟过程，但效率低，适合理解问题
- **递推**：效率高，适合求解，但需要找准状态定义和转移方程

### 2. 状态定义是关键

- 好的状态定义能让转移方程变得简单自然
- 本题中 `(还有几个数没入栈, 栈里有几个数)` 是最符合操作过程的状态

### 3. 正向思维 vs 反向思维

- **正向思维**：从当前状态推向未来，符合操作顺序，不易出错
- **反向思维**：从过去累加到当前，代码简洁，但需要仔细理解

### 4. 本题的本质

本题实际上就是求**卡特兰数**，这是一个经典的组合数学问题。掌握递推方法后，可以进一步学习卡特兰数的数学性质。

### 5. 对于初学者

建议先掌握**正向思维的二维DP**，因为它的思路最符合操作过程，不容易出错。理解后再去看反向思维的代码和卡特兰数公式。
