# P5019 [NOIP 2018 提高组] 铺设道路 解题总结

## 一、题目回顾

**题目描述**：有一条长度为 n 的道路，第 i 块区域下陷深度为 d_i。每天可以选择一段连续区间 [L,R]，要求区间内所有区域深度均大于0，操作后这些区域深度减1。求最少需要多少天将所有区域深度变为0。

**样例**：

text

```
n = 6
d = [4, 3, 2, 5, 3, 5]
输出：9
```

## 二、AC代码

cpp

```
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    int prev, curr;
    cin >> prev;  // 第一个位置深度
    
    int ans = prev;  // 第一个位置需要的天数
    
    for (int i = 1; i < n; i++) {
        cin >> curr;
        if (curr > prev) {
            ans += curr - prev;  // 只累加上升的部分
        }
        prev = curr;
    }
    
    cout << ans << endl;
    return 0;
}
```

时间复杂度：O(n)，空间复杂度：O(1)

## 三、核心思路

### 1. 问题转化

将"每天选一段连续区间"的操作，转化为**统计每个位置相对于前一个位置的深度增量**。

### 2. 关键观察

- 如果 `d[i] ≤ d[i-1]`：处理前 i-1 个位置时，可以顺便把第 i 个位置处理完
- 如果 `d[i] > d[i-1]`：多出来的部分必须单独处理，需要额外天数

### 3. 计算公式

text

```
最少天数 = d[1] + sum(max(0, d[i] - d[i-1])), i从2到n
```

## 四、贪心思想分析

### 贪心策略

从左到右扫描，**每次只处理当前不得不处理的部分**，即只累加深度上升的差值。

### 贪心选择性的体现

1. **局部最优**：遇到上升时，立即将差额计入答案
2. **不考虑未来**：不需要预见后面的深度变化，当前决策就是最优的
3. **充分利用已有操作**：遇到下降或相等时，直接"蹭"前面的操作

### 为什么贪心是对的？

因为深度下降的部分必然能被前面的操作覆盖，而深度上升的部分必然需要新增操作，且这些新增操作无法与前面的操作合并。

## 五、数学归纳法证明

设 `f(i)` 表示处理完前 i 个区域所需的最少天数。

**基础情况**：

- `f(1) = d[1]`：只有一个区域时，需要 d[1] 天 ✓

**归纳假设**：

- 假设 `f(i-1)` 已经是最优解

**归纳步骤**：证明 `f(i) = f(i-1) + max(0, d[i] - d[i-1])`

**分两种情况讨论**：

**情况1：d[i] ≤ d[i-1]**

- 处理前 i-1 个区域需要 d[i-1] 次操作
- 由于 d[i] ≤ d[i-1]，这 d[i-1] 次操作每次都可以延伸到第 i 个区域
- 因此第 i 个区域已经被完全覆盖，不需要新增操作
- 所以 `f(i) = f(i-1)`

**情况2：d[i] > d[i-1]**

- 处理前 i-1 个区域最多只能覆盖第 i 个区域 d[i-1] 次
- 剩余 `d[i] - d[i-1]` 的深度必须由新增操作完成
- 这些新增操作不能与前面的操作合并（因为前 i-1 个区域此时已为0）
- 所以 `f(i) = f(i-1) + (d[i] - d[i-1])`

**综上**：`f(i) = f(i-1) + max(0, d[i] - d[i-1])` 成立 ✓

由数学归纳法，该公式对所有 i 成立，且得到的解是最优的。

## 六、总结与启示

### 1. 思维方式的转变

- **从"过程"到"总量"**：不要纠结每天具体怎么选区间，而是思考每个位置需要被覆盖几次
- **从"全局"到"局部"**：利用相邻关系简化问题，将全局最优转化为局部最优

### 2. 此类问题的特征

- 操作具有连续性（区间操作）
- 存在某种"覆盖"关系
- 可以通过差分或相邻比较来简化
