# ⏱ 二、完整流程（带时间限制）

---

## 🟡 阶段1：思考（上课 / 碎片时间）

- ⏰ 时间：≤ 20分钟

### 内容：
- 理解题意
- 构思解法（暴力 / 枚举 / 数据结构等）
- 在脑中模拟过程

### ❗允许使用AI的条件：
- 卡住 ≥10分钟完全没思路
- 已尝试2种方法仍无进展

👉 AI使用规则：
- 只允许问思路
- 禁止要代码

---

## 🔵 阶段2：编码实现（自习环境）

- ⏰ 时间：≤ 20分钟

### 要求：
- 独立完成代码
- 不依赖AI
- 尽量一次写完整

### ❗强制规范：

```cpp
#include <bits/stdc++.h>
using namespace std;
#define int long long
```

## 🔴 阶段3：Debug（核心能力）

- ⏰ 时间：≤ 30分钟

### Debug流程：

1. 打印变量（cout）
2. 手动模拟样例
3. 检查边界：
   - n = 1
   - 最大数据范围
   - 特殊情况

### ❗允许使用AI的条件：

- Debug ≥30分钟仍无法解决

👉 AI使用方式：

- 只问：“我的代码哪里有问题？”
- 禁止直接要完整解法

------

## 🟢 阶段4：复写（防止依赖AI）

- ⏰ 时间：≤ 15分钟

### 条件：

如果使用过AI（思路 / debug）

👉 必须：

- 关闭AI
- 自己重新完整写一遍代码

------

## 📝 阶段5：题解 + 复盘（最重要）

- ⏰ 时间：≤ 10分钟 / 每题

------

### 每题必须手写以下内容：

## 📌 1. 题解（思路）

- 这题的核心思路是什么？
- 为什么这样做？

------

## ❌ 2. 问题与错误

- 卡在哪？
- 错误类型：
  - 思路问题？
  - 代码实现？
  - 边界？
  - 粗心？

------

## 🔁 3. 反思

- 为什么会犯这个错误？
- 是否是习惯问题？

------

## 🚀 4. 展望

- 下次遇到类似题如何更快解决？
- 是否可以优化？

------

👉 示例：

- 忘记使用 long long
- 边界情况没考虑
- 思路少了一步
