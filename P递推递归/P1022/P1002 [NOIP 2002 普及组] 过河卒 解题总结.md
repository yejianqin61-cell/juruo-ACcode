# P1002 [NOIP 2002 普及组] 过河卒 解题总结

## 一、题目描述

棋盘上A点(0,0)有一个过河卒，需要走到目标B点(n,m)。卒行走规则：只能向下或向右。同时在棋盘上C点有一个对方的马，该马所在的点和所有跳跃一步可达的点称为对方马的控制点（马走"日"字）。要求计算出卒从A点能够到达B点的路径条数，马的位置固定不动。

**输入格式**：一行四个正整数，分别表示B点坐标和马的坐标
**输出格式**：一个整数，表示所有的路径条数
**数据范围**：1 ≤ n, m ≤ 20，0 ≤ 马的坐标 ≤ 20

## 二、AC递归代码（记忆化搜索）

cpp

```
#include <bits/stdc++.h>
using namespace std;

struct dian {
    int x;
    int y;
};

struct dian b, ma;
long long memo[25][25];  // 记忆化数组

// 判断是否为马的控制点
bool is_control(int x, int y) {
    // 马本身的点
    if (x == ma.x && y == ma.y) return true;
    // 马可以跳到的8个点
    if (x == ma.x - 1 && (y == ma.y + 2 || y == ma.y - 2)) return true;
    if (x == ma.x - 2 && (y == ma.y + 1 || y == ma.y - 1)) return true;
    if (x == ma.x + 1 && (y == ma.y + 2 || y == ma.y - 2)) return true;
    if (x == ma.x + 2 && (y == ma.y + 1 || y == ma.y - 1)) return true;
    return false;
}

long long dfs(int x, int y) {
    // 越界或马的控制点
    if (x > b.x || y > b.y || x < 0 || y < 0) return 0;
    if (is_control(x, y)) return 0;
    
    // 到达B点
    if (x == b.x && y == b.y) return 1;
    
    // 记忆化：如果已经计算过
    if (memo[x][y] != -1) return memo[x][y];
    
    // 只能向下或向右
    memo[x][y] = dfs(x + 1, y) + dfs(x, y + 1);
    return memo[x][y];
}

int main() {
    cin >> b.x >> b.y >> ma.x >> ma.y;
    memset(memo, -1, sizeof(memo));
    cout << dfs(0, 0) << endl;
    return 0;
}
```

## 三、递归解法的局限性

1. **递归深度问题**：当n=m=20时，最坏情况下递归深度可达40层，虽然不算太深，但递归调用开销较大
2. **栈空间占用**：每个递归调用都会占用栈空间，大量递归可能导致栈溢出
3. **重复计算**：如果不加记忆化，会有大量的重复子问题计算（原始递归代码会TLE）
4. **效率相对较低**：虽然加了记忆化后时间复杂度O(n×m)，但递归调用的函数开销比循环大
5. **代码可读性**：递归思路直观，但对初学者来说理解递归过程有一定难度

## 四、AC递推代码（动态规划）

cpp

```
#include <bits/stdc++.h>
using namespace std;

struct dian {
    int x;
    int y;
};

int main() {
    struct dian b, ma;
    cin >> b.x >> b.y >> ma.x >> ma.y;
    
    long long dp[25][25] = {0};  // dp[i][j]表示从(0,0)到(i,j)的路径数
    bool control[25][25] = {false};
    
    // 马的控制点偏移量（包括马本身）
    int dx[] = {0, -1, -1, -2, -2, 1, 1, 2, 2};
    int dy[] = {0, -2, 2, -1, 1, -2, 2, -1, 1};
    
    // 标记所有马的控制点
    for (int i = 0; i < 9; i++) {
        int nx = ma.x + dx[i];
        int ny = ma.y + dy[i];
        if (nx >= 0 && nx <= 20 && ny >= 0 && ny <= 20) {
            control[nx][ny] = true;
        }
    }
    
    // 初始化起点
    dp[0][0] = control[0][0] ? 0 : 1;
    
    // 动态规划填表
    for (int i = 0; i <= b.x; i++) {
        for (int j = 0; j <= b.y; j++) {
            // 跳过起点和控制点
            if ((i == 0 && j == 0) || control[i][j]) {
                continue;
            }
            // 状态转移：只能从上面或左面来
            if (i > 0) dp[i][j] += dp[i-1][j];
            if (j > 0) dp[i][j] += dp[i][j-1];
        }
    }
    
    cout << dp[b.x][b.y] << endl;
    return 0;
}
```

## 五、递推解法的局限性

1. **空间占用**：需要二维数组存储所有状态，当n,m较大时（如1000），空间占用会很大
2. **必须按顺序计算**：递推需要按照特定顺序（从上到下，从左到右）计算，不能随意跳跃
3. **状态定义固定**：一旦确定状态定义（dp[i][j]表示到(i,j)的路径数），很难灵活修改
4. **不适合动态变化的问题**：如果问题条件动态变化（如马会移动），递推方法需要重新计算
5. **理解门槛**：对初学者来说，理解状态转移方程需要一定的抽象思维能力

## 六、总结经验

### 1. 算法选择经验

- **小数据范围（n,m≤20）**：递归+记忆化和递推都可以，递归更直观，递推更高效
- **大数据范围**：必须用递推（动态规划），避免递归栈溢出
- **代码竞赛**：优先使用递推，效率更高，不易超时

### 2. 编程技巧总结

- **数据类型选择**：路径数可能很大，必须用`long long`防止溢出
- **边界处理**：始终注意数组下标不要越界（0-20的范围判断）
- **控制点标记**：预先计算并标记所有不能走的点，避免重复判断
- **初始化**：起点的特殊处理很重要（dp[0][0]=1）

### 3. 常见错误避免

- **马的移动方向**：8个方向要写对，不要遗漏马本身的位置
- **递推公式**：搞清楚是从"上"和"左"来，不是"下"和"右"
- **循环范围**：要循环到`<=b.x`和`<=b.y`，确保B点被计算
- **跳过条件**：控制点和起点要正确跳过

### 4. 思维提升

- **从递归到递推**：递归是"从上到下"思考，递推是"从下到上"构建
- **状态定义**：找到合适的子问题定义是解题关键
- **空间换时间**：记忆化和动态规划都是典型的空间换时间策略
- **问题抽象**：过河卒问题本质是带障碍的网格路径计数问题

### 5. 举一反三

这类问题的变种很多，掌握了核心思想后可以解决：

- 不同移动规则（如可以向上向左）
- 多个障碍物
- 带权值的路径
- 三维空间的路径计数
- 带时间维度的动态规划

**最后总结**：过河卒问题虽然简单，但包含了动态规划的核心思想——将大问题分解为相互关联的子问题，通过填表方式避免重复计算。这是算法学习的入门经典，掌握好这个问题对后续学习更复杂的动态规划问题很有帮助！
